# FileHound 结果操作与主窗口自适应设计

日期：2026-04-10  
状态：已确认，待实现

## 1. 背景

当前 FileHound 已经具备基础搜索与结果展示能力，但与目标参考体验相比仍有明显缺口：

- 结果项只能看见，缺少 Finder / Find Any File 风格右键操作
- 多选结果项没有统一动作模型
- 结果视图没有 Finder / QuickLook 风格图标与缩略图
- 主窗口随着条件增加不会自适应高度，也没有在超出屏幕时切换到可滚动条件区
- 主窗口标题仍误用了参考 App 标题，默认宽度也偏大

本轮目标是把“看得见但不能用”的原型，推进到“可以实际管理搜索结果”的桌面工具状态。

## 2. 本轮目标

- 结果窗口三种视图统一支持单选、多选和右键菜单操作
- 右键菜单能力尽量对齐 Finder / FAF 的可见操作集合
- 操作执行后结果列表立即刷新，保持所见即所得
- 结果图标改为 Finder / QuickLook 风格渲染
- 主窗口标题恢复为 `FileHound`
- 主窗口规则区域支持增高、自适应移动和超限滚动
- 主窗口默认宽度进一步收紧，更接近实际可用尺寸

## 3. 范围

### 3.1 包含范围

- 结果窗口上下文菜单
- 多选结果统一动作管线
- Finder / QuickLook 风格图标与缩略图渲染
- 主窗口条件区动态高度与滚动
- 主窗口标题、默认尺寸修正
- 结果项动作后的本地状态刷新

### 3.2 不包含范围

- 新增偏好设置项，例如 `Show Icon Preview`
- 文件操作撤销系统
- 对复杂权限失败场景做完整错误恢复向导
- 自定义文件标签颜色编辑器，仅调用系统支持的标签写入

## 4. 结果菜单设计

### 4.1 菜单结构

三种结果视图统一提供同一套右键菜单。菜单项顺序按参考体验组织：

1. `Open`
2. `Open With`
3. `Reveal in Finder`
4. 分隔线
5. `Move to Trash`
6. `Delete Immediately`
7. 分隔线
8. `Copy Path`
9. 分隔线
10. `Get Info`
11. `Rename`
12. `Create Alias in...`
13. `Quick Look`
14. `Set Label`
15. `Make Invisible` / `Make Visible`
16. `Unlock`
17. 分隔线
18. `Remove from Results`
19. 分隔线
20. `Services`

### 4.2 多选规则

- 菜单对单选与多选统一生效
- `Rename` 仅在单选时可用
- `Quick Look` 支持单选或多选预览
- `Open With` 对多选只显示同时适用于全部选中项的 App
- `Set Label`、`Move to Trash`、`Delete Immediately`、`Remove from Results`、`Make Invisible`、`Unlock`、`Reveal in Finder`、`Copy Path`、`Create Alias in...` 支持多选

### 4.3 行为规则

- `Move to Trash`：直接执行，不弹确认
- `Delete Immediately`：弹确认后永久删除
- `Remove from Results`：只从当前结果投影移除，不影响磁盘文件
- `Rename`：原地进入编辑态或弹单文件重命名输入
- `Set Label`：使用 macOS 标签写入
- `Make Invisible / Make Visible`：切换文件隐藏属性
- `Unlock`：移除锁定属性；若已解锁则菜单项可禁用
- `Open With`、`Services`：动态取系统可用项，不手写固定菜单

## 5. 结果刷新规则

所有动作执行后立即反映到当前结果窗口：

- 删除或移除：对应条目立刻消失
- 重命名：名称、路径、排序投影立即更新
- 标签、隐藏、锁定状态变化：图标和元数据立即刷新
- 多选批量操作：操作完成后整批刷新，不逐条闪烁

若单个条目操作失败：

- 继续执行其余条目
- 聚合显示失败摘要
- 成功项仍正常刷新

## 6. 图标与缩略图

### 6.1 渲染策略

- 表格与树状视图：使用 Finder 风格文件图标
- 网格视图：优先显示 QuickLook / Finder 风格缩略图
- 若缩略图不可用，则回退到系统文件图标
- 自定义文件图标应被保留，而不是统一替换成类型图标

### 6.2 数据来源

- 图标优先通过系统工作区与 QuickLook 管线获取
- 缩略图请求需异步进行，避免阻塞滚动与选择
- 结果单元格需支持缓存，避免滚动时重复同步取图

## 7. 主窗口布局调整

### 7.1 标题与尺寸

- 主窗口标题改为 `FileHound`
- 默认宽度在当前基础上再收紧，不再直接参考 Retina 截图物理像素
- 默认高度保持紧凑，只容纳首屏规则区

### 7.2 条件区动态高度

- 增加条件行时，主窗口优先按内容自动增高
- 若增高后会超出桌面可视区域，先把窗口上边缘移动到当前屏幕可视区域顶部
- 若移动后仍不足以完整容纳全部条件，则条件区切换为滚动容器
- 底部状态栏、搜索按钮、停止按钮始终固定可见

### 7.3 可视区域规则

- 以窗口所在屏幕的 `visibleFrame` 为准
- 多屏环境下只操作当前窗口所在屏幕
- 窗口不应被移动到菜单栏之上或 Dock 之下

## 8. 架构拆分

### 8.1 结果动作层

新增统一动作层，负责：

- 收集当前选中项
- 生成上下文菜单
- 分发文件操作
- 管理批量操作错误
- 通知结果视图模型刷新

三个结果视图不各自实现文件动作逻辑，只负责把选择状态交给统一动作层。

### 8.2 结果图标层

新增图标提供器，负责：

- 系统图标获取
- QuickLook 缩略图获取
- 缓存与失效策略

### 8.3 主窗口尺寸协调层

主窗口控制器负责：

- 根据规则行数量计算目标高度
- 判断是否超出屏幕可视区域
- 在“扩大窗口”与“规则区滚动”之间切换

规则区自身只负责内容布局，不负责窗口移动。

## 9. 测试策略

### 9.1 单元测试

- 结果动作可用性测试
- 多选动作批量执行测试
- 删除 / 重命名 / 移除结果后的投影刷新测试
- 主窗口高度策略测试
- 超出屏幕时切换滚动策略测试

### 9.2 UI 测试

- 结果项右键菜单出现
- 单选与多选时菜单可用性正确
- `Move to Trash` 后条目消失
- `Delete Immediately` 弹确认
- `Rename` 后标题更新
- 增加多条规则后主窗口高度与滚动行为正确

### 9.3 手工验证

- 对照 FAF 检查右键菜单层级与顺序
- 检查 Finder 风格图标是否正确反映文件真实图标
- 检查主窗口在小屏幕与多条件场景下不会把底部按钮挤出可视区域

## 10. 风险与约束

- `Open With` 与 `Services` 依赖系统动态菜单，需要避免阻塞主线程
- 文件标签、隐藏位、锁定位等元数据修改可能受权限限制，必须逐项容错
- QuickLook 缩略图生成必须异步，否则网格滚动会卡顿
- 批量永久删除属于高风险动作，确认文案必须清晰，且默认焦点不能落在破坏性按钮

## 11. 完成标准

- 结果项右键菜单在三种视图下都可用
- 单选、多选动作行为一致
- 结果操作后列表实时刷新
- 图标与缩略图明显接近 Finder / QuickLook 风格
- 主窗口标题为 `FileHound`
- 增加条件时主窗口先增高，再在必要时切换到滚动条件区
- 默认主窗口宽度较当前进一步收紧
