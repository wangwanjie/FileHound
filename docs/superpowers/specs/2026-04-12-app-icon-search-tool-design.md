# FileHound AppIcon 重设计文档

日期：2026-04-12  
状态：已确认，待实现

## 1. 背景

FileHound 当前已经具备较完整的 macOS 搜索工具骨架，但默认 AppIcon 还没有形成稳定的产品识别。用户已经明确要求重新生成一个更美观的 AppIcon，并替换到项目中。

本轮图标设计方向已经确认：

- 主语义是“搜索”
- 图标气质偏“工具锐利”，而不是温和或通用型
- 图标中必须出现明显可识别的放大镜元素

目标不是做一枚抽象品牌图形，而是让用户在 Dock、Launchpad、Finder 和 DMG 安装场景中，第一眼就把 FileHound 识别为“专业的文件搜索工具”。

## 2. 目标

- 为 FileHound 提供一套新的 macOS AppIcon 资产
- 图标语义明确表达“文件搜索工具”
- 图标在 16 / 32 / 128 / 256 / 512 / 1024 尺寸下都保持可辨识
- 保持 macOS 图标常见的圆角方板基底与专业工具气质

## 3. 非目标

- 不同时重做应用内 UI 主题
- 不引入新的品牌字体、品牌规范或营销视觉系统
- 不修改 `AccentColor` 或应用运行时配色逻辑
- 不在本 change 内处理 DMG 背景图、appcast 封面图或 GitHub Release 宣传图

## 4. 视觉方向

### 4.1 主方案

采用“锐利放大镜 + 文件切片命中”的方向：

- 底形使用 macOS 风格圆角方板
- 主体使用一枚斜放、结构清晰、边缘偏硬朗的放大镜
- 镜片内部出现被扫描到的文件页 / 搜索结果条目 / 命中高亮切片
- 不使用可爱化或过度品牌拟人化的图形

这样可以同时满足三点：

1. 一眼识别为搜索工具
2. 保留“文件搜索”而非通用 Web 搜索的语义
3. 在小尺寸下仍能靠大形体保留识别度

### 4.2 色彩策略

图标色彩采用偏冷的专业工具方向：

- 主色建议为冷青蓝 / 蓝绿
- 辅助高亮使用少量明亮荧光绿或亮青色，表达“命中”
- 避免直接模仿系统 Spotlight 的黑底彩光做法
- 避免大面积高饱和暖色，以免偏离专业桌面工具气质

### 4.3 小尺寸策略

16px 与 32px 是核心风险点。小尺寸下应主动删减细节，仅保留：

- 圆角底板
- 放大镜外轮廓
- 镜片中的大块文件/命中结构
- 一条最关键的高亮命中线

不追求在小尺寸保留完整材质纹理，而是优先保证形体识别。

## 5. 资产交付方式

### 5.1 交付内容

项目最终需要替换 `FileHound/Assets.xcassets/AppIcon.appiconset` 中全部现有 PNG：

- `16x16`
- `16x16@2x`
- `32x32`
- `32x32@2x`
- `128x128`
- `128x128@2x`
- `256x256`
- `256x256@2x`
- `512x512`
- `512x512@2x`

实现阶段建议先生成一个高分辨率主图，再导出各尺寸位图，避免各尺寸分别手工绘制导致风格漂移。

### 5.2 文件命名与工程接入

- 保持现有 `AppIcon.appiconset/Contents.json` 结构不变
- 仅替换各 PNG 文件内容，不额外改动 asset slot 定义
- 若实现阶段需要新增中间产物，应放在临时目录，不提交到仓库

## 6. 实现决策

### 6.1 使用单一主图驱动所有尺寸导出

推荐先确认 1024 级主图，再导出各尺寸，而不是逐尺寸独立设计。

原因：

- 更容易保持视觉一致性
- 后续若继续调整品牌方向，返工成本更低
- 更适合 AppIcon 这种需要多尺寸一致投放的资产

备选方案：

- 每个尺寸单独微调甚至重绘
- 不采用，因为会大幅增加工作量，而且容易造成风格不统一

### 6.2 保持 asset catalog 接入方式稳定

本轮只替换图标资源，不改 `Contents.json` 的槽位组织。

原因：

- 项目已经有标准 macOS AppIcon 结构
- 风险最低
- 不会引入额外的 Xcode 资源配置变更

### 6.3 用人工审查 + 构建验证双重验收

验收不只看图片本身，还要确认：

- Xcode / `xcodebuild` 不出现缺失 icon slot 的警告
- Finder、Dock、应用切换器里的小尺寸辨识度达标

## 7. 风险与缓解

- 图标细节过多导致 16/32 尺寸糊成一团
  - 缓解：从设计阶段就限制大形体数量，只保留放大镜、文件切片、命中高亮三个层级
- 放大镜语义过强，变得像泛搜索而不是文件搜索
  - 缓解：在镜片内部加入文件页 / 结果条目结构，而不是纯抽象高亮
- 与系统 Spotlight 过于相似
  - 缓解：避开 Spotlight 常见的黑底彩光方案，采用 FileHound 自己的冷色工具感方向

## 8. 完成标准

- `AppIcon.appiconset` 所有尺寸位图已被新图标替换
- 图标视觉方向符合“搜索感 + 工具锐利 + 明显放大镜”
- 小尺寸下仍可识别放大镜与命中结构
- 构建流程不出现缺失图标资源的警告
