# FileHound 重构设计文档

日期：2026-04-10  
状态：已确认，待评审

## 1. 背景

当前 FileHound 工程已经具备一部分搜索骨架与窗口原型，但主窗口、结果窗口、偏好设置窗口与目标参考 App 相比仍存在明显差距：

- 界面结构只完成了低保真骨架，未达到 1:1 复刻标准
- 搜索交互、结果浏览和偏好设置的状态管理仍然粗糙
- 现有实现把布局、规则编辑、搜索执行、结果开窗混杂在少数控制器里，继续叠加功能会快速失控

本轮重构的首要目标不是补齐所有历史 spec 功能，而是以“截图可见功能 + 更现代的桌面工具视觉”为第一阶段交付标准，先把用户直接感知到的核心体验做对。

## 2. 本轮目标

本轮重构聚焦以下结果：

- 主窗口在布局、控件顺序、状态切换上高保真复刻参考 App
- 结果窗口补齐默认视图、表格视图、树状视图三种展示
- 偏好设置遵守项目 Apple 规范，使用 segmented 风格 Preferences Window，同时尽量保留原版单页设置内容密度
- 统一窗口视觉语言，采用更现代的冷石墨底色与高识别交互蓝
- 将 UI 层按模块拆分，建立清晰状态机和可维护边界

## 3. 本轮范围

### 3.1 包含范围

- 主搜索窗口完整重构
- 搜索范围选择菜单重构
- 搜索条件行与逻辑展示重构
- 搜索中与空闲态的状态切换
- 结果窗口工具栏与三视图重构
- 结果内过滤与排序入口重构
- 偏好设置窗口重构
- 主题 / 语言热切换接入新窗口结构
- 搜索会话取消、结果窗复用、基础错误容忍

### 3.2 明确降级到后续阶段

以下能力不作为本轮完成标准：

- 特权 Helper 完整可用化
- Full Disk Access 深度引导优化
- 保存搜索 / 最近搜索完整工作流
- 权限诊断页的完整交互体验
- 更复杂的内容搜索能力扩展
- 全量旧 spec 中未直接体现在参考截图里的高级能力

这些能力不被否定，只是从本轮目标中降级，避免 UI 重构阶段范围失控。

## 4. 视觉与交互原则

### 4.1 复刻原则

- 主体布局、控件顺序、信息密度严格跟随参考 App
- 视觉升级只作用于配色、阴影、材质和细节质感
- 不改变参考 App 的使用模型，不把专业工具改成互联网产品式大卡片界面

### 4.2 新视觉方向

- 主背景使用冷石墨色系，弱化原版偏旧的塑料感
- 高亮与主要交互沿用电蓝色，但更干净、更锐利
- 列表高亮、按钮激活、segment 选中统一为同一语义色
- 工具栏按钮默认不加明显背景边框，仅在选中时给出高亮

### 4.3 可用性约束

- 所有输入框、popup、按钮文案默认水平居中
- 搜索中的 loading 固定出现在主按钮左侧，垂直居中
- 结果窗口搜索框与工具条图标保持垂直居中，不可超出窗口右边界
- 偏好设置窗口顶部使用真正的 segmented 风格，而不是四个独立按钮

## 5. 主窗口设计

### 5.1 窗口结构

主窗口保持接近参考图的紧凑横向布局，默认尺寸约为 `1140 × 300`，结构分为三段：

1. 范围选择行  
   `Find Items [scope popup] where`
2. 条件面板  
   包含加减按钮、字段 popup、操作符 popup、值输入框、条件逻辑摘要
3. 底部状态行  
   左侧为状态文案，右侧为 `Find / Stop` 主按钮

### 5.2 搜索范围选择

范围菜单保留原版语义，分层组织为：

- 预设范围
  - on startup volume
  - on all disks
  - on local disks
  - on network volumes
  - in Finder selection
  - inside folder...
- Recent Locations
- 当前已挂载卷列表

范围菜单在结构上使用原生菜单模型，不做自定义复杂浮层。

### 5.3 条件编辑器

条件编辑器首版支持参考图中直接可见的编辑方式：

- 多条规则行
- 字段 popup
- 操作符 popup
- 文本值输入
- 加一条 / 删除一条
- 条件逻辑摘要展示

字段与操作符的完整枚举以参考图为准。当前已存在的字段与操作符模型会保留，但 UI 会重新组织。

### 5.4 状态切换

主窗口包含三个明确状态：

- `idle`
  - 尚未开始搜索，允许完整编辑
- `editing`
  - 条件已改动但未执行，允许完整编辑
- `searching`
  - 冻结范围选择、规则编辑、加减按钮和逻辑切换
  - 主按钮从 `Find` 切换为 `Stop`
  - loading 指示器出现在主按钮左侧
  - 左下角显示 `Searching: <scope>` 与动态命中数

搜索结束或取消后回到 `editing` 或 `idle`，保留最后的命中统计。

## 6. 结果窗口设计

### 6.1 总体结构

结果窗口采用单窗口多视图模型，默认尺寸接近参考图，包含：

- 顶部原生风格工具栏
- 中部结果展示区域
- 右上角过滤搜索框
- 右下角固定 `matched` 计数

### 6.2 顶部工具栏

工具栏固定包含以下分组：

- View
  - 默认图标视图
  - 表格视图
  - 树状视图
- Invisibles
  - 显示 / 隐藏
- Package Contents
  - 显示 / 隐藏
- Trashed
  - 显示 / 隐藏
- Filter 搜索框

工具按钮默认保持轻量，仅在选中态显示高亮，不使用明显按钮边框容器。

### 6.3 三种结果视图

#### 默认视图

- 网格化图标预览
- 顶部提供 Preview Size 控件
- 顶部提供 Sort By 控件

#### 表格视图

- 列定义接近参考图：
  - Name
  - Kind
  - Date Modified
  - Size
- 命中词在名称列中高亮显示
- 支持按列排序

#### 树状视图

- 目录层级缩进接近 Finder / FAF 原生表达
- 复用与表格视图一致的列定义
- 可折叠目录节点

### 6.4 过滤与排序

- 右上角 Filter 只作用于当前结果投影，不触发新搜索
- 三视图共享同一份过滤文本
- 排序控件采用原生 `NSMenu` 风格弹出菜单
- 排序字段至少包含：
  - Name
  - Date Modified
  - Date Created
  - Last Opened
  - Date Added
  - Kind
  - Size
  - Tags
  - Enclosing Folder
  - Path

### 6.5 结果窗口复用

- 如果结果窗口已打开，再次搜索时复用该窗口
- 复用时刷新标题、结果集、当前视图投影和统计信息
- 不重复创建无限个结果窗口

## 7. 偏好设置设计

### 7.1 结构约束

严格遵守 `VanJay Apple 平台开发规范`：

- 使用 Preferences Window
- 固定包含 `Appearance` 和 `Updates`
- 窗口不可由用户拖动缩放
- 各分段宽度一致
- 内容区使用滚动容器
- 切换时窗口高度可动画变化，但 segmented 控件位置不偏移

### 7.2 顶部分段

偏好设置顶部使用单一的 segmented 风格分段控件，本轮固定四个分段：

- General
- Search
- Appearance
- Updates

### 7.3 各分段内容

#### General

- Hot Key 设置
- 仅在 Finder 中工作 / 全局工作
- Open Recent Search 菜单
- 应用窗口关闭行为

#### Search

- Expand all folders when showing results
- Show Results Early
- Include Spotlight results
- Special Folders 入口

#### Appearance

- Theme popup
- Language popup
- Font Size 窄输入框 + 右侧上下 stepper
- Dim Color 预览色块 + 点击选择区域
- 其他结果外观相关偏好

#### Updates

- 检查更新策略 popup，仅包含：
  - 启动时检查
  - 手动检查
  - 每天自动检查
- 自动下载更新开关
- 立即检查更新
- 恢复默认设置

### 7.4 更新相关窗口

- `Updates` 分段负责策略与偏好
- Sparkle 的独立更新面板继续作为外部弹出窗口使用
- 偏好设置页中的 `Check Now` 触发更新检查，而不是自己复刻 Sparkle 详情面板

## 8. 架构拆分

### 8.1 现有问题

当前 `SearchSplitViewController` 同时负责：

- 布局
- 规则编辑
- 搜索计划组装
- 目录遍历调用
- 内容匹配调用
- 结果窗口打开

这会让 UI 变更和搜索逻辑耦合，导致重构成本持续上升。

### 8.2 新的模块边界

本轮拆分为以下模块：

- `SearchWindowController`
  - 主窗口生命周期
- `SearchFormViewController`
  - 主窗口表单、状态栏、`Find/Stop`
- `SearchScopeMenuProvider`
  - 搜索范围菜单数据
- `SearchRuleListController`
  - 规则行管理与逻辑展示
- `SearchWorkflowController`
  - 搜索触发、取消、结果流转
- `ResultsWindowController`
  - 结果窗口生命周期
- `ResultsToolbarController`
  - 工具栏状态与操作
- `ResultsCollectionViewController`
  - 默认图标视图
- `ResultsTableViewController`
  - 表格视图
- `ResultsOutlineViewController`
  - 树状视图
- `PreferencesWindowController`
  - 分段与窗口动画
- `GeneralPreferencesViewController`
- `SearchPreferencesViewController`
- `AppearancePreferencesViewController`
- `UpdatesPreferencesViewController`

### 8.3 搜索内核接入策略

本轮不推翻已有搜索引擎模块，而是通过 `SearchWorkflowController` 把现有能力整理为稳定入口：

- 组装 UI 查询
- 发起搜索会话
- 接收结果流
- 接收进度与统计
- 处理取消
- 将结果推送给结果窗口

## 9. 状态与数据流

### 9.1 单一结果源原则

默认视图、表格视图、树状视图共享一份基础结果源，任何视图切换都只改变投影，不重新创建独立结果模型。

### 9.2 共享状态

结果窗口共享以下状态：

- 当前视图模式
- 是否显示 invisibles
- 是否显示 package contents
- 是否显示 trashed
- 当前 filter 文本
- 当前排序字段

### 9.3 偏好设置热更新

- 主题切换立即作用于主窗口、结果窗口、偏好设置窗口
- 语言切换立即刷新文案
- 字体大小与列表外观偏好变更后，结果窗口可立即重绘

## 10. 错误处理与边界行为

- 无权限目录：跳过并记录统计，不中断整次搜索
- 搜索取消：立即终止遍历与匹配任务，不保留悬挂状态
- 结果过滤 / 排序：仅改变当前结果投影，不重跑搜索
- 结果窗口已存在时重复搜索：复用窗口
- 偏好设置切换分段：只切换内容区，不销毁整个窗口
- 窗口布局必须保证 0 约束冲突

## 11. 测试策略

### 11.1 单元测试

- 主窗口状态机测试
- 结果过滤与排序状态测试
- 偏好设置持久化测试
- 范围菜单数据构造测试

### 11.2 UI 测试

- 主窗口启动与基本元素存在
- `Find` 到 `Stop` 的状态切换
- 结果窗口三视图切换
- 结果窗口过滤输入
- 偏好设置 segmented 切换
- 主题 / 语言热切换基础验证

### 11.3 手工验证

- 主窗口与参考图对照
- 结果窗口三视图与参考图对照
- 偏好设置与参考图对照
- 检查更新策略 UI 与参考图对照

## 12. 分阶段实施顺序

建议按以下顺序落地：

1. 重建主窗口结构与状态机
2. 重建结果窗口与三视图工具栏
3. 重建偏好设置 segmented 窗口
4. 将主题 / 语言 / 偏好持久化接入新结构
5. 整理搜索工作流与结果窗口复用
6. 补齐 UI 测试与 smoke test

## 13. 成功标准

当满足以下条件时，本轮可视为完成：

- 主窗口与参考图在结构和状态切换上高度一致
- 结果窗口三视图均可用，工具栏行为稳定
- 偏好设置采用 segmented 风格并满足 Apple 规范
- 主题 / 语言热切换生效
- 搜索会话支持取消，结果窗口支持复用
- 构建通过，关键 UI 测试通过
