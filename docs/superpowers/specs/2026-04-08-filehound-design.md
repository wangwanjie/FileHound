# FileHound 设计文档

日期：2026-04-08
状态：已确认，待进入实现计划

## 1. 产品定位

FileHound 是一款面向高级用户的 macOS 独立分发文件搜索工具，目标能力尽量接近 Find Any File。

它不依赖 Spotlight，不预建全局索引，而是基于用户当前查询实时扫描文件系统，并在搜索过程中持续输出结果。产品重点不是“快速打开一个搜索框”，而是提供一套专业、可持续使用的桌面搜索工作台。

第一版采用独立分发模式，不以 Mac App Store 为目标。应用将支持普通权限搜索、特权搜索模式，以及 Full Disk Access 引导，以最大化可搜索范围。

## 2. 第一版目标

第一版要交付的是“完整桌面工具版”的主体能力，而不是最小可行原型。核心目标如下：

- 支持复杂文件搜索条件组合
- 支持纯文本内容搜索与正则匹配
- 支持列表视图与树形视图两种结果浏览方式
- 支持实时扫描和流式结果输出
- 支持普通权限与特权搜索模式切换
- 支持 Full Disk Access 检测与引导
- 支持保存搜索、最近搜索和默认搜索模板
- 支持浅色 / 深色主题热切换
- 支持简体中文、繁体中文、英文热切换
- 支持 Sparkle 自动更新

## 3. 第一版明确包含的能力

### 3.1 查询能力

- 文件名包含、不包含
- 完整路径包含、不包含
- 扩展名是、不是
- 文件类型筛选：文件、文件夹、别名、符号链接、包
- 文件大小筛选：大于、小于、区间
- 创建日期与修改日期区间筛选
- 可见 / 隐藏文件筛选
- 名称正则匹配
- 文本内容包含匹配
- 文本内容正则匹配
- 搜索范围控制：指定目录、全盘
- 是否搜索隐藏文件
- 是否进入包内容
- 是否跟随符号链接
- 规则分组：全部满足、任一满足、排除规则

### 3.2 结果能力

- 列表视图
- 树形层级视图
- 按列排序
- 结果内二次过滤
- 多选
- 在 Finder 中显示
- 打开文件
- 打开所在目录
- 复制路径
- 文本命中上下文片段预览
- 命中原因展示
- 搜索进度、命中数、已跳过目录数展示

### 3.3 工作流能力

- 保存搜索
- 最近搜索
- 默认启动搜索模板
- 偏好设置窗口
- 权限诊断与修复入口

## 4. 第一版暂不包含的能力

以下能力不纳入第一版范围：

- 压缩包内部内容搜索
- docx、xlsx、pdf 等复杂文档正文解析
- 已删除但未清空文件搜索
- AppleScript 自动化
- CLI 接口
- 全局快捷键呼出搜索窗口
- 结果导出报表

这些能力会显著增加首版复杂度，但不影响先交付接近 Find Any File 的主体体验。

## 5. 产品信息架构

### 5.1 主窗口

主窗口是用户的持续工作区，采用多分栏结构：

- 顶部：快速搜索栏、执行控制按钮、权限模式切换
- 左侧：已保存搜索、最近搜索、默认模板
- 中央上部：高级规则构建器
- 中央下部：结果区
- 右侧：预览与检查面板
- 底部：扫描进度、命中统计、权限状态、错误摘要

主窗口应支持分栏调整，但默认布局必须稳定，不依赖额外弹窗承载主要搜索功能。

### 5.2 结果区

结果区固定支持两种模式：

- 列表视图：用于排序、批量浏览和快速过滤
- 树形视图：按目录层级展示命中结果，帮助理解目录上下文

两种视图共享同一批搜索结果数据，只改变展示投影，不创建两套独立结果模型。

### 5.3 右侧预览与检查面板

选中结果后展示以下信息：

- 文件名、路径、大小、类型、修改时间
- 命中原因
- 文本命中片段
- 权限状态
- 快捷操作入口

### 5.4 偏好设置窗口

偏好设置窗口采用 macOS Preferences Window 形式，固定包含以下 Tab：

- 外观
- 更新
- 搜索
- 权限

要求如下：

- 各 Tab 宽度一致
- 内容区使用滚动容器
- 切换 Tab 时窗口高度动画变化
- Tab Bar 位置不偏移
- 窗口不可由用户拖拽缩放

## 6. 查询系统设计

### 6.1 双层查询结构

查询系统采用“快速查询栏 + 高级规则构建器”的双层结构。

快速查询栏用于处理常见关键字搜索；高级规则构建器负责表达复杂查询逻辑。这样既能支持专业场景，又不会让简单查询变得笨重。

### 6.2 规则构建方式

每条规则为一行，支持：

- 新增
- 删除
- 启用 / 禁用
- 拖拽调整顺序
- 自然语言摘要展示

规则分组支持：

- 满足全部规则
- 满足任一规则
- 排除以下规则

### 6.3 文本内容搜索策略

文本内容搜索作为显式规则存在，而不是默认把关键词同时用于文件名和正文搜索。这样可以避免结果噪音过高，并让性能预期对用户保持透明。

正文搜索支持：

- 普通字符串匹配
- 正则匹配
- 大小写控制
- 常见文本编码尝试

## 7. 搜索结果交互设计

### 7.1 列表视图列定义

列表视图建议默认展示以下列：

- 名称
- 路径
- 大小
- 种类
- 修改时间
- 命中原因
- 权限状态

### 7.2 流式结果更新

搜索过程中必须逐步输出结果，不等待全量扫描完成。结果区应支持边扫描边切换列表 / 树形视图，且保持当前排序、过滤和选择状态尽可能稳定。

### 7.3 结果命中解释

每一条结果都应能回答“为什么命中”。例如：

- 名称命中
- 路径命中
- 内容命中
- 多规则联合命中

这项能力是复杂搜索体验中的关键，否则用户很难理解结果来源。

## 8. 系统架构

### 8.1 总体结构

采用三层架构：

- 主 App
- Search Engine Service
- Privileged Helper

主 App 负责界面、查询编辑、结果展示、预览、保存搜索、偏好设置、更新和权限引导。

Search Engine Service 负责规则编译、执行计划生成、目录遍历、元数据过滤、内容搜索、结果流式输出、取消控制和统计汇总。

Privileged Helper 负责在需要提权时执行受控的文件系统访问能力。

### 8.2 单一搜索逻辑原则

普通模式与特权模式必须共用一套搜索引擎逻辑，只切换底层文件访问 Provider。不能维护两套搜索实现，否则后期会出现行为漂移与结果不一致问题。

### 8.3 内部模块划分

- QueryCompiler：将规则树编译为内部谓词模型
- SearchPlanBuilder：根据查询决定执行策略
- DirectoryWalker：高性能目录遍历
- MetadataEvaluator：基于路径与元数据快速过滤
- ContentMatcher：对候选文本文件执行正文匹配
- ResultStreamer：分批向 UI 推送结果与进度
- SearchSession：管理搜索生命周期、取消状态和统计信息
- FilesystemAccessProviding：抽象普通访问与特权访问

## 9. 权限模型

### 9.1 普通权限模式

在普通权限下，应用搜索当前用户可访问的文件范围，不依赖管理员授权。

### 9.2 特权搜索模式

当搜索涉及受普通权限限制的目录时，应用可切换到特权搜索模式。该模式通过 Privileged Helper 提供更高权限的文件系统访问能力。

### 9.3 Full Disk Access

为了尽量接近全盘搜索，应用需要检测并引导用户授予 Full Disk Access。

必须明确：

- 特权 Helper 并不等于自动拥有全部受 TCC 保护目录访问权
- Full Disk Access 是扩大搜索可见范围的必要组成部分
- 产品文案需准确表达“普通权限 + 特权 Helper + FDA 引导”的组合关系

### 9.4 权限失败处理

权限错误不能中断整个搜索流程，应作为可见警告记录到结果统计中，例如：

- 被权限跳过的目录数量
- 典型失败路径示例
- 当前模式下的限制说明

## 10. 搜索执行模型

搜索执行分为三个阶段：

### 10.1 规划阶段

将用户查询编译为执行计划，决定：

- 搜索根目录
- 是否需要正文搜索
- 是否进入包内容
- 是否跟随符号链接
- 是否需要特权访问

### 10.2 遍历阶段

按目录流式遍历文件系统，并优先使用元数据规则快速剪枝，尽早淘汰不可能命中的路径。

### 10.3 精查阶段

仅对通过前置过滤的候选文件执行正文读取与正则匹配，避免无差别读取大量文件内容。

## 11. 稳定性与性能要求

### 11.1 稳定性要求

- 搜索必须可立即取消
- 关闭窗口后后台会话必须正确结束
- 坏符号链接、循环目录、不可读文件、超长路径都要安全降级
- 权限失败不得导致整个搜索崩溃

### 11.2 性能策略

- 优先进行低成本元数据过滤
- 文本内容搜索仅针对候选文件执行
- 对超大文本文件设置大小阈值
- 文本读取采用分块策略
- 结果按批推送，避免 UI 高频抖动

## 12. 工程约束

根据项目规范，第一版采用以下技术约束：

- 语言：Swift
- UI：AppKit
- 响应式逻辑：Combine
- 布局：SnapKit 优先
- 轻量存储：MMKV
- 依赖管理：SPM 优先
- 自动更新：Sparkle
- 最低系统版本：macOS 11.0
- 默认支持语言：简体中文、繁体中文、英文
- 默认支持浅色 / 深色和跟随系统

## 14. 测试策略

第一版至少建立以下三层测试：

### 14.1 查询与规则单元测试

验证规则编译、布尔组合、大小比较、日期比较、正则逻辑和排除规则行为。

### 14.2 搜索引擎集成测试

基于测试夹具目录验证：

- 普通目录遍历
- 隐藏文件
- 包内容搜索
- 符号链接处理
- 正文搜索
- 取消行为
- 权限失败降级

### 14.3 UI 自动化测试

覆盖：

- 新建搜索
- 编辑规则
- 运行和停止搜索
- 切换结果视图
- 保存搜索
- 权限提示展示
- 预览面板联动

## 15. 发布与分发

第一版采用独立分发模式：

- DMG 分发
- Sparkle 自动更新
- 提供特权 Helper 安装 / 修复入口
- 首次启动提供权限说明与 FDA 引导

## 16. 首版范围结论

第一版交付的是一个专业、可持续使用的 macOS 文件搜索桌面应用主体，重点对齐 Find Any File 的以下体验：

- 实时文件系统搜索
- 强查询表达能力
- 列表 / 树形结果浏览
- 专业结果解释
- 特权模式与权限引导

第一版不追求覆盖所有极端场景，而是优先建立正确的产品骨架、正确的权限模型和正确的搜索架构，为后续复杂文档解析、自动化接口和更多高级能力打下稳定基础。
